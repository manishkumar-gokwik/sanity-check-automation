🔍 Sanity Check Automation — Complete Flow

🚀 Trigger

User clicks "Run Sanity Check" on dashboard (localhost:5000) OR daily 5 PM scheduler
↓

📊 PHASE 0: Load Merchant Data Google Sheets API

0.1 Read "Manish sanity" sheet → Merchant names, EB MID, Expected rates (UPI/CC/DC)
0.2 Read "Salt n key" sheet → Expected KEY + SALT per merchant
0.3 Read "Auto Check" tracker → Current check status
↓

🌐 PHASE 1: Easebuzz Partner Portal Browser Automation

URL: partners.easebuzz.in
1.1 Login → Email/Password from .env
1.2 Navigate: Custom Reports → Merchant Settlements
1.3 Click "Generate New Report" → Fill form (All merchants, last 30 days)
1.4 Wait 60-90s for report generation
1.5 Download CSV → settlement_report.csv
↓

✅ Per-Merchant Checks (1-3)

1 Settlement

Source:
settlement_report.csv
Logic:
Filter by EB MID → count transactions + amount
PASS: Active WARN: No txns

2 MDR

Formula:
(Service Charge / Amount) × 100
Compare:
Actual vs Expected from sheet
PASS: Match (±0.05%) FAIL: Mismatch

3 Account Number

EB Account:
settlement_report.csv
Cheque:
Drive OCR Tesseract → regex
PASS: Match FAIL: Mismatch (fraud!)
↓

🌐 PHASE 2: GoKwik Dashboard Browser Automation

URL: dashboard.gokwik.co
2.1 Login → Email/Password from .env
2.2 OTP auto-fetch Gmail IMAP → Extract 6-digit from "GoKwik Signin" email
2.3 Navigate: Kwik Payment → Settings → Terminals
↓

✅ Per-Merchant Checks (4-5)

4 SALT & KEY

Steps:
1. Switch merchant (dropdown)
2. Navigate Terminals again
3. Click Edit on Easebuzz
4. Extract KEY + SALT from form
5. Compare with "Salt n key" sheet
PASS: Match FAIL: Mismatch

5 VPA / Webhook

Steps:
1. Click Orders in sidebar
2. Open first order (KWIK...)
3. Find Payment → Link column
4. Parse upi://pay?pa=...
5. Check merchant name in VPA
PASS: VPA valid WARN: No orders
↓

📋 PHASE 3: Results

3.1 Aggregate: Overall = PASS / FAIL / WARN per merchant
3.2 Display on dashboard (localhost:5000 → Results tab)
3.3 Write back to "Auto Check" sheet
3.4 Export: PDF / CSV / Email Draft for discrepancies
3.5 Daily 5 PM automatic run via APScheduler

📚 Data Sources Summary

  • Google Sheets API "Manish sanity" — Merchant list + Expected MDR rates
  • Google Sheets API "Salt n key" — Expected SALT & KEY per merchant
  • Google Sheets API "Auto Check" — Tracker (results written here)
  • Browser partners.easebuzz.in — Settlement Report (actual txn data)
  • Browser dashboard.gokwik.co — Production SALT/KEY + VPA from orders
  • Drive API Cancelled Cheque folder — Bank account via OCR
  • Gmail IMAP paymentsops@gokwik.co — Auto OTP fetch

🎯 Why Each Check?

1. Settlement
Verify settlements are actually happening
2. MDR
Ensure correct rates are charged (commercial team validation)
3. Account Number
Detect fraud — wrong bank = money going to wrong person
4. SALT & KEY
Ensure production config matches agreed credentials
5. VPA / Webhook
Verify payment routing — merchant's UPI ID correctness
